Wiederholung: Verschiedene Garbage Kollektoren

Seriell GC

  • Einfacher und effizienter Kollektor
  • Für kleine Heaps und einzelne CPU-Kerne
  • Stop-the-world Kollektor
  • Verwendet Mark-Sweep-Compact Algorithmus

Parallel GC

  • Standardkollektor für Server-Anwendungen
  • Nutzt mehrere Threads für GC
  • Optimiert für Durchsatz
  • Stop-the-world, aber schneller als Seriell GC

Garbage First GC (G1)

  • Für große Heaps (>4GB) konzipiert
  • Teilt Heap in Regionen auf
  • Priorisiert Regionen mit viel Garbage
  • Ziel: Vorhersagbare Pausenzeiten

Concurrent Mark Sweep (CMS)

  • Deprecated seit Java 9
  • Entfernt in Java 14
  • G1 als Ersatz

Epsilon Garbage Collector

  • Do nothing GC
  • Keine Garbage Collection
  • Fährt runter, wenn kein Speicher mehr frei
  • Super geringe Latenz

Shenandoah

  • Ähnlich zu G1
  • Kann Objekte während Applikation läuft verschieben
  • Deutlich schnellere GC
  • Deutlich intensiver für CPU

ZGC

  • Geringe Latenzzeit und Wartezeiten
  • Applikation läuft während GC weiter
  • Leichte Benutzbarkeit
